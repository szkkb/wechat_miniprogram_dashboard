# Practice Prompts | 实践指导提示词

> 使用说明：这些提示词用于具体的实践行动。帮助你将学到的理论转化为实际行动。

---

## 周度实践节奏

### PP-001 | 周度 Loop 检查

```markdown
请帮我进行本周的 Growth Loop 健康检查。

**本周数据**：
（请在使用时填写）

- 新增客户数：
- 客户来源分布：
  - 销售主动开发：___ 个
  - 老客户推荐：___ 个
  - 线上询价：___ 个
  - 其他（说明）：___ 个
- 本周服务的货量：
- 客户反馈（好评/投诉）：
- 本周尝试的增长动作：

**请帮我分析**：

1. **Loop 运转状态**
   - 各获客渠道的表现如何？
   - 哪个渠道效率最高？
   - 有没有什么异常信号？

2. **转化率分析**
   - 询价到成交的转化率如何？
   - 新客户到复购的转化情况？
   - 推荐客户的占比变化？

3. **本周亮点和问题**
   - 做得好的地方
   - 需要关注的问题
   - 潜在的机会

4. **下周行动建议**
   - 最重要的 1-2 件事
   - 需要跟进的客户或动作
   - 可以尝试的小实验
```

### PP-002 | 增长实验复盘

```markdown
请帮我复盘这个增长实验。

**实验基本信息**：
- 实验名称：
- 实验假设：
- 实验时间：
- 投入资源：

**实验结果**：
- 主要指标结果：
- 次要指标结果：
- 预期 vs 实际对比：
- 意外发现：

**请帮我分析**：

1. **结果判断**
   - 实验成功还是失败？依据是什么？
   - 假设被验证还是被推翻？

2. **原因分析**
   - 为什么会产生这样的结果？
   - 哪些因素是关键的？
   - 有没有干扰因素影响了结果？

3. **学习收获**
   - 我们学到了什么？
   - 这对我们理解 Loop 有什么启发？
   - 有没有意外的洞察？

4. **下一步行动**
   - 基于这个实验，下一步应该做什么？
   - 需要进一步验证什么？
   - 可以直接规模化的是什么？

5. **实验方法改进**
   - 这个实验设计有什么可以改进的？
   - 下次类似实验应该注意什么？
```

## 客户增长实践

### PP-101 | 推荐机制设计

```markdown
请帮我设计一个适合[你的行业] 公司的客户推荐机制。

**我的业务情况**：
- 现有客户数量：
- 客户类型：
- 平均客单价：
- 客户复购周期：
- 目前有没有推荐奖励：

**请帮我设计**：

1. **推荐机制框架**
   - 推荐奖励的形式（现金/服务折扣/积分/其他）
   - 奖励的时机（推荐时/成交时/复购时）
   - 奖励的金额/幅度
   - 双向奖励还是单向奖励

2. **推荐流程设计**
   - 客户如何发起推荐？
   - 被推荐人如何联系我们？
   - 如何追踪推荐来源？
   - 奖励如何兑现？

3. **推广策略**
   - 如何让客户知道这个机制？
   - 什么时机提醒客户推荐最合适？
   - 需要什么话术和物料？

4. **效果预测**
   - 预期的推荐率是多少？
   - 需要达到什么转化率才能盈利？
   - 需要追踪的指标

5. **试点计划**
   - 建议从哪些客户开始试点？
   - 试点时间和规模
   - 成功标准
```

### PP-102 | 内容获客策略

```markdown
请帮我设计一个适合[你的行业] 公司的内容获客策略。

**我的情况**：
- 目标客户是谁：
- 客户常用的信息渠道：
- 目前有没有做内容：
- 可投入的时间/人力：

**请帮我设计**：

1. **内容定位**
   - 我们应该输出什么类型的内容？
   - 内容的核心价值主张是什么？
   - 与竞品内容的差异化在哪里？

2. **内容主题规划**
   - 客户最关心的问题是什么？（列出 10-20 个主题）
   - 按客户决策阶段分类这些主题
   - 优先级排序

3. **渠道选择**
   - B2B 物流内容应该发布在哪些渠道？
   - 不同渠道的内容形式调整
   - 渠道优先级

4. **内容到客户的转化路径**
   - 读者如何变成询价客户？
   - 需要设置什么转化钩子？
   - 后续跟进流程

5. **执行计划**
   - 内容生产频率建议
   - 最小可行的启动方案
   - 需要的工具和资源

6. **效果衡量**
   - 追踪什么指标？
   - 多久评估一次效果？
   - 优化迭代的方向
```

### PP-103 | 销售 Loop 优化

```markdown
请帮我优化销售团队的获客 Loop。

**当前情况**：
- 销售团队人数：
- 月均新增客户数：
- 主要获客方式（陌拜/电销/展会/转介绍等）：
- 平均成交周期：
- 询价到成交转化率：

**请帮我分析和优化**：

1. **当前 Loop 诊断**
   - 现有销售流程可以描述为一个 Loop 吗？
   - 如果是，这个 Loop 的结构是什么？
   - Loop 中最弱的环节是什么？

2. **让销售工作形成 Loop 的方法**
   - 如何让每次销售都产生下一次销售的线索？
   - 成交客户如何反哺新客户获取？
   - 销售知识如何沉淀和复用？

3. **关键环节优化建议**
   - 线索获取环节可以怎么优化？
   - 转化效率可以怎么提升？
   - 成交后的动作如何设计？

4. **可量化的改进指标**
   - 当前状态的关键数据
   - 优化目标
   - 如何追踪改进效果

5. **具体行动计划**
   - 第一个月可以尝试什么？
   - 需要销售团队配合做什么改变？
   - 需要什么工具或系统支持？
```

## 客户留存实践

### PP-201 | 复购提升策略

```markdown
请帮我设计提升客户复购的策略。

**当前情况**：
- 活跃客户数：
- 平均复购周期：
- 复购率：
- 客户流失的主要原因（如果知道）：

**请帮我分析和设计**：

1. **客户生命周期分析**
   - 客户从首单到流失的典型路径
   - 关键的留存节点在哪里？
   - 什么信号预示客户可能流失？

2. **复购驱动因素**
   - 客户为什么会持续选择我们？
   - 可以增强哪些因素？
   - 有什么阻碍复购的因素需要消除？

3. **主动触达策略**
   - 什么时机应该主动联系客户？
   - 用什么内容/理由触达？
   - 触达的频率和渠道

4. **客户分层运营**
   - 如何对客户进行分层？
   - 不同层级客户的运营策略差异
   - 资源投入优先级

5. **具体行动计划**
   - 立即可以执行的 3 件事
   - 需要建设的能力或系统
   - 效果衡量方法
```

## 数据追踪实践

### PP-301 | 增长指标体系搭建

```markdown
请帮我设计一个适合[你的行业] 公司的增长指标追踪体系。

**我的情况**：
- 目前有使用什么系统（CRM/ERP/Excel）：
- 目前追踪哪些数据：
- 数据记录的频率和方式：

**请帮我设计**：

1. **核心指标选择**
   - 最应该追踪的 5-8 个核心指标是什么？
   - 每个指标的定义和计算方法
   - 指标之间的关系

2. **数据收集方法**
   - 每个指标需要什么原始数据？
   - 如何收集这些数据？
   - 记录频率建议

3. **指标看板设计**
   - 建议的看板结构
   - 日/周/月各看什么指标？
   - 如何可视化？

4. **预警机制**
   - 每个指标的健康范围
   - 需要关注的异常信号
   - 发现问题后的响应流程

5. **简化起步方案**
   - 如果只能追踪 3 个指标，选哪 3 个？
   - 最简单的实现方式（比如用 Excel）
   - 如何逐步升级追踪能力
```
